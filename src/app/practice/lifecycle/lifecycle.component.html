Lifecycle of component devided into two :

    1. For Parent components (Parent lifecycle executes before child )
  • ngOnChanges

    Called Whenever Input (Data bound) value changed (@Input)

  • ngOnInit

    Called only once when component gets loaded.

    Use to initialize data in component

    Called after input values are set when a component initialized

    Added by default to every component by default by angular CLI

    Called after ngOnChanges()

  • ngDoCheck

    Whenever something changes on the template of component or inside the component

    Called during every change detection (update/detect/change in Properties etc..) run

    Called after ngOninit()

    2. For Parent components
  • ngOnContentInit
  • ngOnContentChecked
  • ngOnViewInit
  • ngOnViewChecked
  • ngOnDestroy
